# Agent 设计思路汇报

这一组文档面向**第一次接触本项目的工程师**，回答的是「这个 Agent 为什么长这样」——心智模型、原理、关键取舍，而不是逐行 API。需要精确签名与数据结构时，请到 [ddd/](../ddd/) 查（本目录与 DDD 互补、不复制内容）。

## 与其它文档的关系

| 文档 | 回答的问题 | 与本目录的关系 |
|---|---|---|
| [prd/](../prd/) | 要做成什么、验收标准 | 需求来源 |
| [ddd/](../ddd/) | 怎么实现（架构、模块签名、数据流，§1–§35） | **规格**；本目录讲「为什么」，遇到细节链接回它 |
| [plan/01plan.md](../plan/01plan.md) · [plan/02plan.md](../plan/02plan.md) | 分几步做、何时算完 | 实施路线 |
| **本目录** | 为什么这么设计、新人如何上手与扩展 | **设计思路 + 扩展实践** |

## 一句话定位

> 它是一个**从零手写的最小可用 ReAct Agent**：以「**主循环 + 生命周期中间件**」为骨架——借鉴 LangGraph 的运行时生命周期与 LangChain 的中间件抽象，但它是 **Agent 本体，不是造 Agent 的框架**，因此没有 graph/node 抽象。
