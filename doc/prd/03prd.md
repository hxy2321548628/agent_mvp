# 第三阶段需求（2026-06-26 新增）

> 在二期（[02prd](02prd.md) R1–R9）交付后，面向「从 MVP 到可用产品」补充的 9 项需求 R10–R18。
> 设计落点见 [03ddd](../ddd/03ddd.md) §24–§33，分阶段计划见 [plan/03plan.md](../plan/03plan.md)。
> 定位转变：从「单用户、窗口串行、内存态」转向「评测可回归、多会话并发、持久化 + 分层记忆、性能分级」。

## R10 结构化可观测（评测前置）
- 把「一次 run」沉淀为机读结构化轨迹（JSONL）：每轮模型决策、工具选择、各阶段耗时、token usage（含缓存命中）、估算成本。
- 与现有「日志（人读审计）/ trace（stdout 调试）」分工：Observe 面向机器 / 评测。

## R11 评测回归体系
- 建立针对 **Agent 行为** 的评测集与回归，指标：工具选择准确率 / 任务成功率 / 平均轮数 / token 成本 / 时延。
- **混合判定**：录制回放（确定性、零成本、进 CI）守骨架 + 少量 `@slow` 真实 API 冒烟补真实波动。
- 接入 CI 闸门：回归不过不许合并。

## R12 会话持久化
- 会话状态从进程内 dict 落地为**本地 JSONL**（每会话一份、追加写），进程退出不丢、可重启续聊。
- 目录按**项目路径转义**隔离（借鉴 Claude Code）；序列化保留 `Message` 子类型（判别联合）。

## R13 src 异步核心
- 将 `src` 升级为**纯异步核心库**（LLM / 中间件 / 工具 / runtime / agent / session 全 async），支撑多会话并发与未来 web / 飞书前端。
- 消费端会话天然隔离（CLI 不同 session 不同进程、web 前端会话页面隔离），不存在 sync/async 混用的「染色之痛」。

## R14 并行工具调用
- 一条 AIMessage 含多个 tool_call 时**并发执行**（`asyncio.gather`），缩短多工具轮时延。
- 副作用闸门：HITL 授权串行、执行并行；写类工具按策略串行，避免竞态。

## R15 Prompt 缓存 + 上下文压缩升级
- **缓存**：稳定前缀（system + tools schema）置顶且顺序固定，提升 DeepSeek 自动前缀缓存命中；观测命中率验证。
- **压缩**：从「按消息条数的破坏性摘要」升级为「按 **token 预算** 触发的**非破坏**压缩」——完整历史落盘、只压上下文视图。

## R16 分级模型路由
- 按任务复杂度路由模型档位：小模型 / 启发式判路由与简单任务、大模型负责复杂生成，省成本降时延。
- 路由层对上层透明（仍是 `LLMClient`）；路由决策记入可观测轨迹。

## R17 分层记忆 + 项目级长期记忆
- **分层**：工作记忆（当前上下文）/ 情景记忆（跨会话事件）/ 语义记忆（提炼事实）。
- **项目级 + 渐进式披露**（借鉴 Claude Code）：索引常驻（仅 description，省 token），完整 body 按相关度 / `recall` 工具**按需载入**。
- 语义 / 情景记忆按**相关度**（向量检索 top-k）召回，而非按时间。

## R18 上线前安全与健壮性（接 web / 飞书前）
- 单请求级**取消 / 中断**（状态一致）；工具**沙箱**（替代「正则 + HITL」的弱隔离）。
- **多用户**会话隔离 + 鉴权 + 限流；新增第二个 `LLMClient` 实现（多 provider）验证抽象与各家缓存策略差异。

## 验收
- 沿用 [plan/01plan.md](../plan/01plan.md) §2 / [plan/03plan.md](../plan/03plan.md) §2 的全局完成定义（TDD 全绿、触及代码覆盖率 ≥ 80%、`ruff` 干净、函数 ≤ 50 行、关键参数集中 `config.py`、单数命名、真实外部 `@slow`）。
- **行为回归**：凡改动主循环 / 中间件 / LLM 客户端，必须先过 R11 评测集（录制回放）零行为回归再合并。
- 各需求「在什么情况下 → 期望什么」的用例见 [plan/03plan.md](../plan/03plan.md) 各阶段（P15–P23）。
